# MUSE 221e aligned quaternion dataset

This folder is generated by `export_muse_221e_dataset.m`.

Each trial is exported after 221e compensation, quaternion SLERP resampling, onset-based alignment, and robot-to-MUSE frame transformation.

Main per-trial file: `trajectories/<trajectory>/<axis>/aligned_quaternions.csv`.

Quaternion order is `[w x y z]`. The correction quaternion is defined as `q_err = conjugate(q_muse) * q_robot`, so the robot orientation can be reconstructed as `q_robot ~= q_muse * q_err`.

Euler angles use ZYX yaw-pitch-roll convention in degrees. The exported Euler-error vector is ordered as `[roll_x, pitch_y, yaw_z]`.

Rows containing NaN/Inf time or quaternion components, or near-zero quaternion norm, are removed before export. The manifest and per-trial metadata report `n_raw_samples`, `n_removed_samples`, and `valid_fraction`. Trials below the configured validity threshold are exported but marked as questionable.

Inspection plots are written under `Inspection/` when `cfg.write_inspection_plots = true`. These PNG files are only for sanity checking alignment and error behavior; they are not required for AI training.
